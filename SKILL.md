# 微信自动化技能

## 1. Description（技能详细说明）

这是一个微信客户端自动化控制工具，可以唤醒微信窗口、截图、识别界面内容、模拟点击和输入操作、发送消息、检查新消息。

**注意：微信需要用户手动打开，工具不会自动启动微信。**

**核心特性：**
- 自动唤醒微信窗口（Ctrl+Alt+W 快捷键，仅当微信在托盘区时）
- 所有操作前自动激活窗口，确保窗口在前台
- 每次操作都获取最新截图，不使用缓存
- 支持多窗口标题匹配（"微信"、"WeChat"、"Weixin"）
- **OCR结果由AI解析，不再在代码中固定解析，适应不同窗口大小**

## 2. When to use（触发场景）

当用户说以下内容时，使用此技能：

- "打开微信"、"唤醒微信"、"显示微信"
- "帮我截个微信的图"、"微信截图"
- "微信界面上有什么内容"、"识别微信界面文字"
- "点击微信上的XXX"、"在微信里输入XXX"
- "发送消息"、"在当前聊天发送XXX"
- "滚动微信聊天记录"
- "帮我操作微信"相关任务
- "微信状态"、"微信是否在运行"
- **"看看谁给我发消息了"、"查看聊天列表"、"检查新消息"** → 使用 `get_chat_list` 或 `check_new_messages`
- **"查看和XXX的聊天记录"、"看看和XXX的聊天"、"读取XXX的聊天内容"** → 使用完整流程（见第5.1节）
- **"给XXX发消息"、"和XXX说话"、"告诉XXX"** → 使用完整流程（见第5.2节）
- **"回复XXX"** → 使用 `auto_reply`

### 3.10 get_chat_list - 获取聊天列表（OCR原始结果）

**功能：** 获取微信窗口的OCR识别原始结果，由AI自行分析联系人和消息

**调用：**
```bash
python {baseDir}/agent.py get_chat_list
```

**参数说明：**

|| 参数名 | 类型 | 必需 | 默认值 | 说明 |
||--------|------|------|--------|------|
|| --count | int | 否 | 5 | 保留参数（兼容旧接口） |

**返回值：**

成功：
```json
{
  "status": "success",
  "data": {
    "ocr_results": [
      {
        "text": "识别的文本",
        "confidence": 0.95,
        "position": {"x": 100, "y": 200},
        "box": [[x1,y1], [x2,y2], [x3,y3], [x4,y4]]
      }
    ],
    "window_rect": {"left": 0, "top": 0, "width": 800, "height": 600},
    "total": 10
  }
}
```

**重要说明：**
- 不再在代码中解析联系人和消息，直接返回原始OCR结果
- AI需要根据坐标位置自行判断哪些是联系人、哪些是消息内容
- `position` 是左上角坐标，`box` 是四个角的坐标

**AI分析OCR结果的规则：**

```
【微信窗口布局分析】

1. 左侧聊天列表区域（X坐标较小，通常 < 窗口宽度的30%）
   - 联系人名称：通常在X坐标 50-200 范围
   - 消息预览：在联系人名称下方或右侧
   - 时间：X坐标较大（通常在右侧）
   - 未读数量：可能显示为红色数字

2. 右侧聊天窗口区域（X坐标较大，通常 > 窗口宽度的30%）
   - 聊天窗口标题：显示当前聊天对象名称（在顶部）
   - 聊天记录：多条消息按时间排列
   - 消息气泡：我发的在右侧，对方发的在左侧
   - 输入框：底部区域

【关键判断逻辑 - 重要！】

1. 区分左侧聊天列表和右侧聊天窗口
   - 通过X坐标判断：X < window_width * 0.3 → 左侧列表
   - X > window_width * 0.3 → 右侧聊天窗口

2. 右侧聊天窗口的消息归属判断（重点！）
   - 聊天窗口顶部显示当前聊天对象名称
   - 该窗口内的所有消息都属于这个联系人
   - 如果右侧没有聊天窗口（只有列表），则无法判断
   - ⚠️ 注意：右侧可能显示多条聊天记录，需要根据Y坐标区分不同消息
   - ⚠️ 群聊中不同人的消息需要根据发送者名称区分

3. 区分"我发的消息"和"对方发的消息"
   - 我发的消息：X坐标较大（通常在窗口右半部分）
   - 对方发的消息：X坐标较小（通常在窗口左半部分）
   - 消息气泡位置是关键判断依据

4. 时间识别
   - 格式：HH:MM、昨天、前天、日期等
   - 通常X坐标较大，在消息右侧或列表右侧

5. 聊天记录匹配（重要！）
   - 右侧聊天窗口可能显示多条聊天记录
   - 需要根据Y坐标区分不同消息
   - 同一联系人的消息按Y坐标从小到大排列（时间从早到晚）
   - 群聊中不同人的消息需要根据发送者名称区分
```

**用途：**
- AI分析OCR结果，识别联系人列表
- AI判断哪些联系人有新消息
- AI计算点击坐标，用于后续操作

### 3.13 check_new_messages - 检查新消息（OCR原始结果）

**功能：** 获取微信窗口OCR结果，供AI判断是否有新消息

**调用：**
```bash
python {baseDir}/agent.py check_new_messages
```

**参数说明：** 无参数

**返回值：**

```json
{
  "status": "success",
  "data": {
    "ocr_results": [
      {
        "text": "识别的文本",
        "confidence": 0.95,
        "position": {"x": 100, "y": 200},
        "box": [[x1,y1], [x2,y2], [x3,y3], [x4,y4]]
      }
    ],
    "window_rect": {"left": 0, "top": 0, "width": 800, "height": 600},
    "total": 10,
    "is_first_init": false
  }
}
```

**返回字段说明：**
|| 字段 | 说明 |
||------|------|
|| `ocr_results` | OCR识别的原始结果列表 |
|| `window_rect` | 微信窗口的位置和大小 |
|| `total` | OCR结果数量 |
|| `is_first_init` | 是否是首次初始化（用于定时任务） |

**用途：**
- AI分析OCR结果，判断是否有新消息
- AI识别联系人名称和消息内容
- AI计算点击坐标用于回复

**AI判断新消息的逻辑：**

```
【分析步骤】

1. 获取窗口宽度，计算分界线
   - 分界线X = window_width * 0.3

2. 分离左侧列表和右侧聊天窗口
   - X < 分界线 → 左侧聊天列表
   - X > 分界线 → 右侧聊天窗口

3. 分析左侧聊天列表
   - 按Y坐标分组（相近的Y坐标属于同一联系人）
   - 每组中找到联系人名称和消息预览
   - 判断消息来源（我发的/对方发的）

4. 判断是否有新消息
   - 对比上次保存的状态（通过联系人名称匹配）
   - 如果消息内容变化且是对方发的 → 新消息

5. 计算点击坐标
   - 找到目标联系人的Y坐标范围
   - 点击坐标：X = window_left + 列表宽度/2, Y = window_top + 偏移 + 联系人Y中心

【聊天记录匹配规则】

右侧聊天窗口的消息归属：
1. 找到窗口顶部的联系人名称 → 这是当前聊天对象
2. 窗口内的所有消息都属于这个联系人
3. 区分消息发送者：
   - 右侧气泡（X坐标大）= 我发的
   - 左侧气泡（X坐标小）= 对方发的

群聊场景：
1. 每条消息可能有发送者名称
2. 根据"发送者名称: 消息内容"格式识别
3. 需要区分是群消息还是私聊消息
```

## 5. 常见操作流程

### 5.1 查看聊天记录流程

**场景：用户说"查看和XXX的聊天记录"、"看看和XXX的聊天"**

**标准流程：**
```
步骤1: 获取OCR结果
  → 调用 get_chat_list() 获取原始OCR结果

步骤2: AI分析OCR结果
  → 根据X坐标区分左侧列表和右侧聊天窗口
  → 在左侧列表中找到目标联系人名称
  → 计算联系人点击坐标

步骤3: 点击联系人进入聊天窗口
  → 调用 click_contact --x 坐标 --y 坐标
  → ⚠️ 注意：点击后需要验证是否成功进入聊天窗口

步骤4: 验证聊天窗口状态（重要！）
  → 点击联系人后，再次调用 get_chat_list() 获取OCR结果
  → 检查右侧是否出现聊天窗口
  → 如果右侧没有聊天窗口，说明可能已经在该聊天窗口，点击反而关闭了
  → 如果确认聊天窗口被关闭，再次点击联系人重新打开

步骤5: 查看聊天记录
  → AI分析OCR结果，根据Y坐标区分不同消息
  → 区分我发的消息（右侧）和对方发的消息（左侧）
  → 或通过 scroll() 滚动查看更多历史记录
```

---

### 5.2 发送消息流程

**场景：用户说"给XXX发消息"、"和XXX说话"、"告诉XXX"**

**标准流程：**
```
步骤1: 获取当前界面状态
  → 调用 get_chat_list() 获取OCR结果
  
步骤2: AI分析当前状态
  → 判断当前是否已在目标聊天窗口
  → 检查右侧聊天窗口顶部是否显示目标联系人名称
  
步骤3: 根据判断结果执行不同操作

  情况A - 已经在目标聊天窗口：
    → 直接跳到步骤5发送消息
    → ⚠️ 不要点击联系人，否则会关闭窗口！

  情况B - 不在目标聊天窗口：
    → 执行步骤4，点击联系人进入聊天窗口

步骤4: 点击联系人并验证窗口状态（关键！）
  → AI计算联系人坐标
  → 调用 click_contact --x 坐标 --y 坐标
  → 再次获取OCR结果验证窗口状态

步骤5: 确认窗口打开后发送消息
  → 调用 send_message --message "内容"
```

```
  → 区分左侧列表和右侧聊天窗口

  → 调用 auto_reply --x 坐标 --y 坐标 --message "回复内容"
```

## 6. 重要机制说明

### 6.1 OCR结果解析机制（核心！）

**设计理念：**
- 微信窗口大小可能变化，固定坐标解析不可靠
- OCR返回原始结果，由AI智能分析
- AI根据相对位置关系判断界面元素

**AI解析OCR的关键规则：**

```
【区域划分】

窗口宽度 = window_rect.width
分界线X = window_width * 0.3

左侧聊天列表：X < 分界线
右侧聊天窗口：X > 分界线

【左侧列表解析】

1. 按Y坐标分组
   - Y坐标相近（差值 < 40像素）的文本属于同一联系人
   - 每组从上到下排列

2. 每组内的元素
   - 名称：X坐标最小的主要文本
   - 消息预览：名称下方的文本
   - 时间：X坐标最大的文本

3. 过滤规则
   - "搜索" 等UI元素应忽略
   - 纯数字可能是未读数量或时间

【右侧聊天窗口解析】

1. 窗口标题
   - 位于右侧顶部区域
   - 显示当前聊天对象名称

2. 聊天消息
   - 按Y坐标从上到下排列
   - 我发的消息：X坐标较大（右侧）
   - 对方发的消息：X坐标较小（左侧）

3. 群聊特殊处理
   - 每条消息前可能有发送者名称
   - 格式："发送者名称: 消息内容" 或 "发送者名称 消息内容"

【点击坐标计算】

联系人点击坐标：
- X = window_rect.left + 列表宽度 * 0.5
- Y = window_rect.top + 联系人Y中心 + 顶部偏移

列表宽度 ≈ window_width * 0.28
顶部偏移 ≈ window_height * 0.08
```

### 6.2 定时监控消息的正确使用方式

**⚠️ 极其重要：定时任务触发时必须先初始化状态！**

**使用 cron.add 工具设置定时任务：**

```
【第一次启动监控时】必须初始化！

步骤1: 重置状态
  → 调用 reset_contact_states()
  
步骤2: 初始化当前状态
  → 调用 check_new_messages()
  → AI记录当前所有联系人状态
  
步骤3: 设置定时任务
  → 使用 cron.add 工具设置下次检查时间
  → schedule.at = 当前时间 + 间隔时间（ISO 8601 格式）
  → payload.kind = "agentTurn"
  → payload.message = "检查微信新消息"

【定时任务触发时】必须先检查状态！

步骤1: 检查状态是否存在
  → 调用 get_contact_states()
  → 如果为空：初始化后结束，不回复消息
  
步骤2: 获取OCR结果
  → 调用 check_new_messages()
  → 返回 is_first_init 字段
  
步骤3: 如果 is_first_init=true
  → 本次是初始化，不回复消息
  → 设置下一次定时任务
  → 结束

步骤4: AI分析OCR结果
  → 判断是否有新消息
  → 如果有，计算坐标并回复

步骤5: 设置下一次定时任务
```

### 6.3 自动回复规则

**模型检测到新消息后，必须自动回复，遵循以下规则：**

```
【回复内容生成规则】

1. 能办到的事情 → 执行并回复结果
   - 查询信息（天气、价格、新闻等）→ 调用相应工具查询后回复结果
   - 简单计算 → 计算后回复答案
   - 设置提醒 → 设置后确认

2. 办不到的事情 → 明确告知原因
   - 需要物理操作 → "抱歉，我无法进行物理操作"
   - 超出AI能力范围 → "抱歉，这个我暂时做不到"

3. 闲聊/问候 → 自然回复
   - "你好" → "你好，有什么可以帮你的吗？"
   - "在吗" → "在的，请说"

4. 不明确的消息 → 礼貌询问
   - "您说的我不太理解，能详细说明一下吗？"
```

## 7. 注意事项

1. **坐标系统**：所有坐标均为屏幕绝对坐标，左上角为 (0, 0)
2. **OCR解析**：由AI根据相对位置智能分析，不再固定解析
3. **窗口大小**：适应不同窗口大小，通过相对比例判断位置
4. **消息发送**：`send_message` 会自动定位输入框，无需手动点击
5. **窗口前置**：所有操作都会自动将微信窗口置顶
6. **无缓存**：每次 OCR 和截图都获取最新画面
7. **窗口状态判断**：点击联系人后必须验证聊天窗口是否打开
8. **流程完整性**：操作需要遵循完整的流程步骤
9. **状态持久化**：联系人状态保存在 `wechat_contact_states.json` 文件中
10. **定时任务初始化**：设置定时任务时必须先初始化状态
